## 0. 使用方式

这份文档不是项目管理用的“大表格”，而是**面向 AI 编程的分阶段开发工单清单**。

使用原则：

1. **一次只让 Claude 实现一个小阶段或 1~3 个连续任务**
2. **先跑通主链路，再补测试、优化、增强**
3. **每完成一个任务就要求 Claude 自测、补单测、更新 README**
4. **不允许 Claude 一次性生成完整平台全部代码**
5. **遇到设计冲突，以“更简单、可运行、可测”为准**

## 1. 总体开发约束

### 1.1 技术约束
- 后端：Python 3.12 + FastAPI + Pydantic v2 + SQLAlchemy 2
- 前端：Next.js + TypeScript + Tailwind + shadcn/ui
- 数据库：PostgreSQL + pgvector
- 缓存/队列：Redis
- 网页读取：httpx + selectolax，失败再 Playwright
- 文件处理：pypdf / python-docx / openpyxl / python-pptx
- 观测：structlog + OpenTelemetry
- 流式输出：SSE

### 1.2 架构约束
- V1 只做**单智能体**
- V1 工具仅保留：
  - web_search
  - web_reader
  - file_parse
  - kb_search
  - python_exec
  - cite_answer
- V1 不做：
  - 多智能体
  - MCP 总线
  - 高风险外部写操作
  - 企业级权限中心
  - 复杂浏览器交互编排

### 1.3 代码约束
- 所有输入输出必须有 Pydantic Schema
- 所有工具必须有独立目录、schema、service、test
- 所有 provider 差异必须收口到 adapter 层
- 不允许在业务层散落 `if provider == ...`
- 所有关键流程必须可记录 trace_id / run_id
- 所有失败必须返回结构化错误对象
- 工具原始输出不能直接回灌模型，必须先 reducer

## 3. 开发阶段总览

### Phase 0：项目底座与开发环境
目标：让仓库、运行环境、数据库、基础服务、CI 能跑起来

### Phase 1：会话、消息、流式聊天骨架
目标：先做一个可聊天、可持久化、可流式输出的最小系统

### Phase 2：模型适配层与运行时骨架
目标：统一 DeepSeek / 智谱接口，跑通单智能体控制循环

### Phase 3：网页研究主链路
目标：实现“搜索 -> 读取网页 -> 清洗 -> 证据 -> 引用回答”

### Phase 4：文件解析与知识检索
目标：实现“上传文件 -> 解析 -> 分块 -> 检索 -> 联合回答”

### Phase 5：后台任务与前端研究面板
目标：把长任务转后台，并让前端看到阶段和进度

### Phase 6：质量、评测、可观测与加固
目标：补齐可测试、可观测、可降级、可上线的基础能力

## 7. Phase 3：网页研究主链路

### 完成定义
完成后应满足：
- 输入研究问题
- 自动生成搜索查询
- 搜索得到候选链接
- 读取网页正文
- 生成段落与 citation
- 输出带引用回答

### 任务清单

#### P3-01 Tool Runtime 骨架
**内容**
- 定义 ToolSpec
- 定义 ToolExecutor
- 定义 ToolCallRecord
- 定义 ToolResultReducer

**验收**
- 工具可以统一注册、执行、记录日志

---

#### P3-02 web_search 工具
**内容**
- 输入 schema：
  - query
  - locale
  - freshness
  - site_allowlist
  - top_k
- 输出 schema：
  - results[{title, url, snippet, domain, publish_time}]

**验收**
- 能返回结构化搜索结果
- 同一 query 有超时、重试、错误包装

---

#### P3-03 搜索 query 重写器
**内容**
- 生成 2~3 条搜索词
- 包含：
  - 原查询
  - 更正式查询
  - 官方来源倾向查询

**验收**
- 同一任务可产生多条候选查询

---

#### P3-04 搜索结果去重与排序
**内容**
- URL 规范化
- 域名去重
- 官方来源优先
- 同域限制

**验收**
- 候选链接质量明显优于原始搜索结果

---

#### P3-05 web_reader（lite 模式）
**内容**
- HTTP 获取网页
- selectolax 提取正文
- 去除导航/页脚/广告
- 转 Markdown
- 生成 paragraph list

**验收**
- 静态网页能正确提取正文
- 返回段落编号 `p1/p2/...`

---

#### P3-06 web_reader（browser 回退）
**内容**
- Playwright 打开页面
- 等待加载
- 获取渲染后 HTML
- 复用正文抽取逻辑

**验收**
- 动态页面在 lite 失败时可回退读取

---

#### P3-07 Citation Builder
**内容**
- 为网页段落生成 citation：
  - source_uri
  - title
  - locator
  - fetched_at
  - quote（可选短摘录）

**验收**
- 每段证据都可映射 citation_id

---

#### P3-08 Evidence Store（V1）
**内容**
- 将网页阅读结果规整成 EvidenceItem
- 写入 run 级别证据池
- 支持后续 answer 阶段读取 evidence_ids

**验收**
- Runtime 能持有网页证据集合

---

#### P3-09 cite_answer 工具
**内容**
- 输入 evidence_ids
- 生成：
  - summary
  - key_points
  - risks_or_unknowns
  - citations

**验收**
- 关键结论都带 citation_id
- 证据不足时明确输出“不确定”

---

#### P3-10 深度研究单轮链路
**内容**
- route -> query rewrite -> search -> pick urls -> read -> evidence -> cite_answer
- 不先做复杂循环，只做**单轮受控研究链路**

**验收**
- “研究某产品并给出引用”可跑通

---

#### P3-11 前端引用面板
**内容**
- 回答中显示 citation 标识
- 点击后展开：
  - 标题
  - URL
  - 段落内容
  - 抓取时间

**验收**
